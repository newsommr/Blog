According to Google's transparency report, 96% of pages loaded on Chrome in the United States are using HTTPS.

This is an incredible statistic. Earlier in the decade, it was not as commonplace to have non financial sites using encryption. I would personally attribute a huge help of this adoption rate to Let's Encrypt which is a non-profit who provides free TLS certificates to any site who needs one. Using encryption on your site can improve it in the following ways:
  • Provides integrity. No one else can intercept what your site is sending to your users and modifying it.
  • Prevents other people/providers from reading the communications between your site and your users which adds privacy.

Adding a TLS certificate to your website is easy thanks to Let's Encrypt.
Certbot is a tool created by the EFF and it automates the process of installing the TLS certificate. It also automates the renewal process once the current certificate expires after 3 months. Once you SSH into your server, you can easily install the certificate by typing the following commands:

sudo snap install core; sudo snap refresh core
sudo snap install --classic certbot
sudo ln -s /snap/bin/certbot /usr/bin/certbot
And assuming you are using Apache (which you are if you are in this class) then you just use this and it'll automatically change your apache configuration:
sudo certbot --apache
sudo certbot renew --dry-run (This tests to make sure the auto renewal works correctly.)

And now your site should be visible over a https connection. Try it!
